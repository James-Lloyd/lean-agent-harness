# state/evidence

End-to-end evidence captured by `/verify` and the `e2e-evidence` skill, one subfolder per task id
(e.g. `evidence/TASK-012/`). Screenshots, request/response captures, run logs, sample outputs — the
proof a skeptic would accept that a change actually works, beyond passing unit tests.

Keep it: review and the human look here. Prune stale evidence for shipped-and-merged work during `/gc`.

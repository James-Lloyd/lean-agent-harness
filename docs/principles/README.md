# docs/principles

The engineering norms and reasoning behind the harness. Unlike the other `docs/` folders (which fill
up as the project runs), these are seeded and meant to be read.

- [`harness-philosophy.md`](./harness-philosophy.md) — why every file exists; read this first.
- [`workflow.md`](./workflow.md) — the plan → execute → validate → review → record lifecycle every
  task moves through, and the gate between each phase.
- [`golden-principles.md`](./golden-principles.md) — this project's taste invariants (the single
  source for subjective preferences; humans + linters reference it).
- [`evaluator-rubric.md`](./evaluator-rubric.md) — how the `evaluator` subagent scores a sprint.
- [`sources.md`](./sources.md) — bibliography; which idea came from where.

# 020 — Frontend ↔ Backend API contract (spec)

> Illustrative stub for the worked example. The root and component `CLAUDE.md` files point here as the
> single source of truth for the FE↔BE contract. In a real project this would hold the full endpoint
> contract; the frontend's typed client (`frontend/lib/api/`) is generated from it.

- Status: illustrative
- Owner: example

## Contract (excerpt)
- `GET /api/metrics?range=<7d|30d>` → `200` with `{ "series": [{ "t": <iso8601>, "v": <number> }] }`.
- Errors return a JSON body `{ "error": <string> }` with a `4xx`/`5xx` status — never a bare `500`.

## Acceptance criteria
- [ ] The frontend's typed client matches this schema (regenerated when the schema changes).
- [ ] The backend rejects an invalid `range` with `400 { "error": "invalid range" }`.
- [ ] The cross-cutting Playwright e2e loads the dashboard and renders a metric from a live backend.
